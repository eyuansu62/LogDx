## Possible Root Cause

PyArrow version mismatch or outdated type stubs: the code references pyarrow.compute functions (`is_null`, `if_else`, `equal`, `utf8_length`, etc.) and pyarrow.fs (`FileSystem`) that are either not present in the installed pyarrow version or not properly exported in the type stubs. The error pattern is consistent across both pyright and mypy, ruling out linter bugs and pointing to an upstream pyarrow version/stub incompatibility.
