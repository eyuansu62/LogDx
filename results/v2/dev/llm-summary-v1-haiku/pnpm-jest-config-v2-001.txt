# CI Failure Summary

## Primary Failure

Multiple cascading failures during CI: (1) native module installation errors with permission denied (EACCES) when running `node-gyp rebuild` for `fuse-native` and missing `@unrs/resolver-binding-linux-x64-gnu`, (2) Jest Haste map collision for `has-glob`, and (3) test assertion failure in `test/config/get.ts` expecting a registry URL without a trailing slash but receiving one with it. The job failed with exit code 1.

## Possible Root Cause

Permission errors during native module installation prevent `fuse-native` and `@unrs/resolver-binding-linux-x64-gnu` from being built/installed. Additionally, Jest encounters a Haste map collision for `has-glob`, and a test assertion fails because the registry URL in the output includes a trailing slash that the test does not expect.

- Root cause of permission errors in the CI environment (EACCES) – may relate to CI runner permissions or sandbox restrictions.
- Whether the Haste collision for `has-glob` is a dependency duplication or configuration issue.
- Whether the trailing slash in the registry URL is a real bug or an environment-specific output difference.
- Whether native module installation failure cascades to test execution or is independent.
- CI environment details (runner OS, Node version, pnpm version).
