## Primary Failure

The `docker buildx bake` action failed during the integration-tests build step. The root cause was an HTTP/1.1 502 Bad Gateway error when downloading the Nydus release package (`nydus-static-v2.4.0-linux-amd64.tgz`) via wget from the GitHub Releases server.

## Possible Root Cause

The dragonflyoss/nydus GitHub Releases endpoint is returning 502 Bad Gateway errors, blocking the Dockerfile's wget operation at step 336. This is an upstream service availability issue. Secondary failures (missing coverage reports, missing docker-compose files) are cascading consequences of the primary build failure stopping execution before coverage reports were generated.

## Uncertainties / Missing Context

- Whether the 502 error is transient or persistent at the nydus releases endpoint
- Why buildx backend is probing for docker-compose.yaml, docker-bake.json, docker-bake.override.hcl, and Dockerfile.rego when an explicit build context is provided (L002510–L003404)
- Context cancellation during image fetches (`failed to do request: Get ... context canceled`, L003814–L003815) — whether this is a timeout from the primary 502 error or a separate network issue
- Cache reservation conflict at job end (`Unable to reserve cache with key docker.io--tonistiigi--binfmt-latest-linux-x64`, L003944) — whether this is a race condition with parallel job runs
