# CI Failure Summary

## Primary Failure

NumPy build failure during Meson configuration and C/C++ compilation. The build system cannot locate required linear algebra libraries (MKL, OpenBLAS, BLIS, BLAS, LAPACK) and encounters multiple C/C++ compiler issues: standard math functions (`isfinite`, `isinf`, `isnan`, `signbit`) are undeclared; C++ standard library detection fails (`_LIBCPP_VERSION` undeclared); and thread-local storage and other compiler features do not compile. A partial build that reaches testing segfaults in `test_datetime_nat_argsort_stability` with exit code 245.

## Possible Root Cause

The CI environment lacks required BLAS/LAPACK system libraries (MKL, OpenBLAS, BLIS, BLAS, LAPACK), causing linker failures. Additionally, the C compiler environment is misconfigured: standard C99 math functions are undeclared despite `-lm` being linked, and C++ standard library detection is broken (libc++ headers missing or improperly configured, `_LIBCPP_VERSION` unavailable). The combination prevents successful NumPy compilation; a partial build that reaches testing triggers a segmentation fault in datetime sorting logic.

## Uncertainties / Missing Context

- Exact GCC/Clang version and C++ standard library variant (libc++, libstdc++, MSVCRT) in use
- CI runner image version and whether BLAS/LAPACK libraries are installed
- Why standard C math headers (math.h) are not providing `isfinite`, `isinf`, `isnan`, `signbit`—typically a consequence of missing include or incorrect compiler flags
- Whether the segfault in `test_datetime_nat_argsort_stability` is a direct consequence of the broken build or an independent runtime bug
- Whether Meson build configuration specifies explicit library paths that may be incorrect for the runner environment
