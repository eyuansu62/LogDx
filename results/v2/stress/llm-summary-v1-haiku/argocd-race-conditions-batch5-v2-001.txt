## Possible Root Cause

The Go module cache (`go/pkg/mod/`) contained pre-existing files that blocked tar extraction during cache restoration. This could occur if: (1) a previous build left module files intact without cleanup, (2) the cache key changed but overlapping modules remained on disk, or (3) concurrent cache operations created file conflicts. The tar "Cannot open: File exists" error indicates files exist at the destination paths but tar cannot overwrite them—likely a permissions or file-locking issue.
